# СПЕЦАВТОПРО

Локальный проект сайта компании СПЕЦАВТОПРО с фронтенд-сборкой на `Rspack`.

## Стек

- `Rspack`
- HTML-шаблоны страниц
- Vanilla JavaScript
- Глобальные CSS-стили
- Локальные изображения, шрифты и SVG-иконки

## Команды

```bash
npm install
```

```bash
npm run dev
```

```bash
npm run build
```

```bash
npm run build:dev
```

## Что внутри

- `src/index.html` — главная страница сайта.
- `src/privacy.html` — страница политики конфиденциальности.
- `src/index.js` — точка входа главной страницы.
- `src/privacy.js` — точка входа страницы политики.
- `src/scripts/main.js` — интерактивность сайта.
- `src/styles.css` — общая точка подключения стилей.
- `src/styles/` — CSS-слои проекта.
- `src/assets/images/` — локальные изображения.
- `src/assets/fonts/` — локальные шрифты.
- `dist/` — готовая production-сборка после `npm run build`.
- `.docs/` — документация по архитектуре, ревью и плану рефакторинга.

## Режимы сборки

- `development` — запускает `rspack dev server` на `http://localhost:3000`, отслеживает изменения и обновляет страницу.
- `production` — собирает оптимизированный `dist/` для публикации.

## Важные замечания

- Все медиа, шрифты и иконки, которые используются сайтом, хранятся локально внутри `src/assets`.
- Страница `privacy.html` пока оформлена как рабочая заглушка и требует замены на утвержденный юридический текст перед публикацией.
- Карта на главной странице оставлена как подготовленный placeholder-блок под будущую интеграцию.

## Деплой

Для публикации собранного сайта на FTP-сервер есть npm-скрипт:

```bash
npm run deploy
```

Он выполняет сборку `dist/` и запускает Node‑скрипт `scripts/deploy.js`,
который перебирает файлы и зовёт `curl` для каждой загрузки.  
`curl` есть на macOS, Linux и в Windows 10/11 по умолчанию, поэтому дополнительного
софта ставить не придётся – задача стала действительно кросс‑платформенной.

✔ **Требование:** наличие `curl` в PATH (проверьте `curl --version`).

Поля `FTP_HOST`, `FTP_USER`, `FTP_PASS` и `FTP_DEST` можно переопределять через
переменные окружения, пример:

```bash
FTP_HOST=some.server.com FTP_USER=user FTP_PASS=secret npm run deploy
```

По умолчанию используются значения `vitlsat4.beget.tech`, `vitlsat4_1`,
`izZ*8ePsQArv` и `public_html`.  Директорию назначения (`FTP_DEST`) обычно
оставляют `/public_html`.

С недавнего обновления скрипт автоматически **очищает содержимое удалённого
каталога перед загрузкой**. Это сделано также через `curl` и работает на всех
поддерживаемых платформах, поэтому не придётся руками логиниться и стирать
файлы. В процессе очистки используется FTP-команда NLST; если сервер закрывает
соединение сразу после выдачи списка (код `56`), скрипт всё равно продолжит
удаление — такое поведение встречается на некоторых хостингах.
