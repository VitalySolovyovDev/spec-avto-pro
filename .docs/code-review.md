# Ревью проекта

## Findings

### Высокий приоритет

1. `src/scripts/main.js`
Форма обратной связи отправляется через `mailto:`. Это лучше, чем фиктивный `alert()`, но сценарий все еще ненадежный: зависит от локального почтового клиента пользователя, не дает гарантированной доставки, не интегрирован с CRM и не позволяет прозрачно обрабатывать ошибки.

### Средний приоритет

2. `src/index.html`
Главная страница содержит большой объем статической разметки и inline SVG sprite прямо в шаблоне. Из-за этого файл тяжело поддерживать, читать и изменять: контент, структура и иконки смешаны в одном документе.

3. `src/index.html`
Повторяющиеся секции полностью захардкожены: преимущества, услуги, техника, этапы, отзывы и FAQ. Это увеличивает вероятность несогласованных правок и усложняет контентные обновления.

4. `src/styles.css`, `src/styles/components.css`, `src/styles/sections.css`
Хотя проект стал чище, CSS для главной страницы все еще остается крупным глобальным слоем. При дальнейшем росте сайта это быстро приведет к накоплению неявных зависимостей между секциями.

5. `src/assets/images/*`
Ассеты уже приведены в более безопасный размер, но в проекте все еще нет полноценной стратегии медиа-оптимизации: `webp/avif`, responsive-версий и системного контроля веса изображений.

6. `src/index.html`
Блок карты оформлен как placeholder. Для пользователя это выглядит как подготовленный, но не реализованный функционал.

### Низкий приоритет

7. `package.json`
В проекте нет команд на lint, format, smoke-проверки или элементарные UI-тесты. Сейчас качество проверяется только успешной сборкой.

8. `src/privacy.css` и `src/styles/privacy-page.css`
Выделение стилей страницы политики в отдельный поток уже упрощает проект, но при дальнейшем развитии стоит так же отделить и стили главной страницы по доменным блокам, а не только по типу файла.

## Открытые вопросы

1. Нужен ли реальный PDF лицензии или отдельная страница с разрешительными документами?
2. Будет ли подключаться реальная карта, и если да, какой провайдер выбран?
3. Нужна ли интеграция формы с email, CRM, Telegram или другим каналом обработки лидов?
4. Планируются ли реальные ссылки на соцсети и мессенджеры?

## Что уже стало лучше

- Удалены временные артефакты выгрузки и сравнения.
- Все исходники сайта перенесены в `src/`.
- Удалены зависимости от внешнего runtime и следы старой выгрузки.
- Добавлены локальные шрифты в `src/assets/fonts/`.
- Страница политики отделена в собственный CSS-поток.

## Куда упрощать дальше

1. Вынести повторяющийся контент в отдельный data-layer.
2. Убрать inline SVG sprite из HTML в отдельный локальный файл.
3. Разделить лендинг на более мелкие шаблонные блоки или render-функции.
4. Разнести CSS главной страницы по доменным зонам: layout, hero, cards, form, footer.
5. Добавить минимальную автоматическую проверку ссылок, сборки и формы.
